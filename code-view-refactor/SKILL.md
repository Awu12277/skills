# code-view-refactor

前端代码重构与优化助手，基于 Martin Fowler《重构 改善既有代码的设计》第二版。

## 何时使用

在以下场景触发本技能：

- 用户提及**重构**、**代码优化**或**改进既有代码**
- 要求识别**代码坏味道**（bad smells）
- 要求**拆分过长函数**、**简化条件判断**或**消除重复代码**
- 要求按照《重构》书中的模式优化代码
- 要求审视代码并提出改进建议
- 要求优化 HTML 结构、CSS 选择器或 JavaScript 逻辑

## 核心工作流

### 第一步：识别坏味道

审视提供的代码，使用以下检查清单识别坏味道：

| 类别 | 常见坏味道 |
|------|-----------|
| **函数** | 函数过长（>30 行）、魔法数字/字符串、嵌套条件判断 |
| **对象/类** | 过大类、过长参数列表、重复代码 |
| **HTML/CSS** | 非语义化标签、选择器嵌套过深、冗余样式规则 |
| **JavaScript** | 全局变量污染、回调地狱、使用 `+` 拼接字符串 |
| **通用** | 用注释解释难以理解的代码、死代码、未使用变量 |

### 第二步：应用重构技术

根据识别出的坏味道，从参考文档（`reference/instruction.md`）选择合适的重构技术：

- **函数级别**：提取函数、重命名函数、用多态替代条件表达式
- **对象级别**：封装变量、提取类、引入参数对象
- **HTML/CSS**：语义化标签、CSS 变量、简化选择器
- **异步代码**：Promise 链 → async/await、用 `Promise.all` 并行请求

### 第三步：验证行为不变

始终确保重构不改变外部行为：
- 测试现有功能仍然正常
- 保持函数输入输出一致
- 确认未引入新 bug

## 快速参考

常见坏味道与重构手法对照：

| 坏味道 | 重构手法 |
|--------|----------|
| 函数过长 | 按职责拆分为更小的函数 |
| 重复代码 | 提取为共享函数/模块 |
| 魔法数字 | 定义为命名常量 |
| 嵌套条件 | 使用卫语句（guard clauses） |
| `switch` 链 | 用多态或查表法替代 |
| 字符串拼接 | 使用模板字符串 |
| DOM 嵌套过深 | 简化 HTML 结构 |
