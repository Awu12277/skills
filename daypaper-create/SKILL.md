# 日报生成技能

从 Git 提交记录中提取当天的工作内容，生成结构化的日报。

## 工作流程

### 第一步：获取提交记录

执行以下命令获取当天的提交：

```bash
# 获取当天的提交，包含提交信息和作者
git log --since="12:00am" --pretty=format:"%h %ad %s" --date=short --author="$(git config user.name)"

# 如果想查看所有当天的提交（不限用户）
git log --since="12:00am" --pretty=format:"%h %ad %an %s" --date=short
```

### 第二步：分析提交内容

**需要保留的有意义的提交类型：**
- `feat:` - 新功能
- `fix:` - bug 修复（重要的业务逻辑修复）
- `refactor:` - 重构（代码优化、架构调整）
- `docs:` - 文档更新
- `perf:` - 性能优化

**需要过滤掉的琐碎提交：**
- `chore: 打包测试`、`fix: 打包测试` - 打包相关测试
- `fix: 删除构建产物` - 构建产物清理
- `Merge` - 合并提交
- `chore: 移除未使用的 xxx` - 单纯的依赖清理
- `chore: update lock file` - 锁文件更新

### 第三步：去重与合并

**去重规则：**
1. **相同功能的连续提交** - 合并为一条（如多次 "fix: 修复 xxx bug"）
2. **同类重构** - 合并为一条（如多个文件同时迁移 ::v-deep）
3. **相同模块的改动** - 合并为一条

**合并示例：**
- `fix: 修复页面A的bug` + `fix: 修复页面B的bug` → `[Bug修复] 修复多处页面bug`
- `refactor: 迁移组件A的 ::v-deep` + `refactor: 迁移组件B的 ::v-deep` → `[代码规范优化] 统一迁移 ::v-deep 到 :deep() 语法`

### 第四步：生成日报

使用以下 Markdown 格式输出：

```markdown
# 日报

## 今日工作内容

1. **[类型]** - 描述
2. **[类型]** - 描述
3. **[类型]** - 描述
4. **[类型]** - 描述
5. **[类型]** - 描述

## 工作总结

对今日工作的整体总结。
```

## 输出要求

- 总结 **5 条**有意义的工作内容
- **每条内容必须不重复** - 相同类型的合并，不同类型分开
- 每条内容简洁明了，突出重点
- 优先展示对业务有价值的工作（如新功能、重构优化）
- 忽略琐碎的测试/构建提交

## 类型标签

使用以下标签标注工作类型：
- `[新功能]` - feat 开头的提交
- `[重构优化]` - refactor 开头的提交
- `[Bug修复]` - fix 开头的提交
- `[代码规范]` - docs、style 等开头的提交
- `[性能优化]` - perf 开头的提交
- `[其他]` - 其他有意义的提交

## 示例

**输入提交：**
```
a1b2c3d refactor: 将 ::v-deep 迁移到 :deep() 语法
d4e5f6g fix: 字段名更改
g7h8i9j feat: 添加安全操作规程历史版本管理功能
h8i9j0k refactor: 迁移 Avue 表格到 Element Plus
i9j0k1l fix: 修复 pageNo 没传的问题
```

**输出日报：**
```markdown
# 日报

## 今日工作内容

1. **[新功能]** 添加安全操作规程历史版本管理功能，支持查看历史版本
2. **[重构优化]** 将 Avue 表格组件迁移到 Element Plus，提升 UI 组件标准化
3. **[代码规范优化]** 将 Vue 组件中的 ::v-deep 迁移到标准 :deep() 语法
4. **[Bug修复]** 修复 pageNo 参数未传递的问题，保证分页功能正常
5. **[数据模型优化]** 优化字段命名规范，统一数据模型结构

## 工作总结

今日完成了多个重要功能的开发和代码质量优化工作，包括新功能上线和遗留系统的组件迁移。
```
