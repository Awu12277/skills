# 周报生成技能

从 Git 提交记录中提取一周内的工作内容，生成结构化的周报。

## 工作流程

### 第一步：获取提交记录

执行以下命令获取最近一周的提交：

```bash
# 获取最近一周的提交，包含提交信息和作者
git log --since="7 days ago" --pretty=format:"%h %ad %s" --date=short --author="你的用户名"

# 如果不确定用户名，先查看
git log --since="7 days ago" --format="%an" | sort | uniq
```

### 第二步：分析提交内容

**需要保留的有意义的提交类型：**
- `feat:` - 新功能
- `fix:` - bug 修复（重要的业务逻辑修复）
- `refactor:` - 重构（代码优化、架构调整）
- `feat:` - 新功能开发

**需要过滤掉的琐碎提交：**
- `chore: 打包测试`、`fix: 打包测试` - 打包相关测试
- `fix: 删除构建产物` - 构建产物清理
- `Merge` - 合并提交
- `chore: 移除未使用的 xxx` - 单纯的依赖清理（除非有特殊意义）

### 第三步：按类别分组整理

将提交按以下维度分类：
1. **新功能开发** - feat 开头的提交
2. **代码重构** - refactor 开头的提交
3. **Bug 修复** - fix 开头的提交
4. **代码清理** - chore 开头的提交（挑选有意义的）

### 第四步：生成周报

使用以下 Markdown 格式输出：

```markdown
# 周报

## 本周工作内容

1. **[功能名称]** - 简要描述完成的功能
2. **[重构优化]** - 简要描述重构内容
...

## 工作总结

对本周工作的整体总结，强调成果和价值。
```

## 输出要求

- 总结 **10 条**左右有意义的工作内容
- 每条内容简洁明了，突出重点
- 优先展示对业务有价值的工作（如新功能、重构优化）
- 忽略琐碎的测试/构建提交

## 示例

**输入提交：**
```
a1b2c3d refactor: 将 ::v-deep 迁移到 :deep() 语法
d4e5f6g fix: 字段名更改
g7h8i9j feat: 添加安全操作规程历史版本管理功能
```

**输出周报：**
```markdown
# 周报

## 本周工作内容

1. **[代码规范优化]** 将 Vue 组件中的 ::v-deep 迁移到标准 :deep() 语法，提升代码兼容性
2. **[数据模型优化]** 优化字段命名规范，统一数据模型结构
3. **[新功能开发]** 添加安全操作规程历史版本管理功能，支持查看历史版本
...
```
